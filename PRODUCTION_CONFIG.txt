# WealthSync Production Configuration
# These are automatically applied when PREFERRED_URL_SCHEME=https

PRODUCTION_FEATURES:
  Security:
    - HTTPS only (PREFERRED_URL_SCHEME=https)
    - Secure session cookies (SESSION_COOKIE_SECURE=True)
    - HttpOnly cookies (SESSION_COOKIE_HTTPONLY=True)
    - CSRF protection (SESSION_COOKIE_SAMESITE='Lax')
    - No debug mode (debug=False)
    - Error handling with logging

  Database:
    - PostgreSQL supported via DATABASE_URL
    - SQLite fallback for development
    - Connection pooling enabled
    - Connection health checks (pool_pre_ping=True)
    - Connection recycling (3600s)

  Logging:
    - Rotating file handler (10MB per file, 10 backups)
    - Production log level (INFO)
    - Request logging enabled
    - Error tracking with context
    - Logs directory: ./logs/

  Deployment:
    - Gunicorn 4 workers (sync mode)
    - 120 second timeout for long-running requests
    - Health check endpoint: /health
    - Automatic database migrations (Procfile release phase)
    - Static asset serving via Flask/Gunicorn

  Monitoring:
    - Health check: GET /health (returns database status)
    - Application logs at logs/wealthsync.log
    - Request logging for all endpoints
    - Error logging with tracebacks
    - Startup logging

REQUIRED_ENVIRONMENT_VARIABLES:
  Core:
    - SECRET_KEY: Strong random key (generate: python -c "import secrets; print(secrets.token_hex(32))")
    - PREFERRED_URL_SCHEME: "https"
    - FLASK_ENV: "production"

  Email (for verification, password reset):
    - MAIL_SERVER: "smtp.gmail.com"
    - MAIL_PORT: "587"
    - MAIL_USERNAME: "your-email@gmail.com"
    - MAIL_PASSWORD: "app-specific-password"
    - MAIL_DEFAULT_SENDER: "WealthSync <your-email@gmail.com>"

  OAuth (Google Login):
    - GOOGLE_CLIENT_ID: "from Google Cloud Console"
    - GOOGLE_CLIENT_SECRET: "from Google Cloud Console"

  Database (optional):
    - DATABASE_URL: "postgresql://..." (if not set, uses SQLite)

RENDER_DEPLOYMENT:
  Option 1: Manual Setup
    1. Create web service on Render
    2. Connect GitHub repository
    3. Set environment variables in dashboard
    4. Add PostgreSQL database service (optional)
    5. Deploy

  Option 2: Infrastructure as Code
    1. Use render.yaml in repository
    2. Render automatically provisions based on YAML
    3. Services and databases created with one deployment

  Health Check:
    - URL: /health
    - Returns: {"status": "healthy", "database": "connected"}
    - Interval: 300 seconds (5 minutes)
    - Monitored by Render

DATABASE_MIGRATION:
  Automatic (Recommended):
    - Procfile includes: "release: flask db upgrade"
    - Runs before each deployment
    - No manual migration needed

  Manual (if needed):
    - SSH into Render instance
    - Run: flask db upgrade
    - Or: flask db migrate -m "description"

SCALING_CONSIDERATIONS:
  Free Tier:
    - Single dyno with 4 workers
    - Suitable for <100 concurrent users
    - SQLite or free PostgreSQL

  Production Tier (recommended for public launch):
    - Multiple dynos with load balancing
    - PostgreSQL Standard or higher
    - Automatic scaling based on demand
    - Better performance and reliability

MONITORING_AND_DEBUGGING:
  Logs:
    - Check logs/wealthsync.log locally
    - Render dashboard shows application logs
    - All requests logged in production
    - Errors include full tracebacks

  Health Check:
    - Manually: curl https://<app-name>.onrender.com/health
    - Returns database connectivity status
    - Used by Render for monitoring

SECURITY_BEST_PRACTICES:
  Implemented:
    - No debug output in production
    - Secure session configuration
    - CSRF protection
    - Error messages don't leak sensitive info
    - Secrets stored in environment only
    - Database connection validation

  Additional (recommended):
    - Use Render's managed PostgreSQL
    - Enable automatic backups
    - Use strong SECRET_KEY
    - Monitor logs for suspicious activity
    - Regular security updates

COMMON_ISSUES_AND_SOLUTIONS:
  Issue: "flask db command not found"
  Solution: Run "pip install -r requirements.txt" first

  Issue: "Database connection refused"
  Solution: Check DATABASE_URL is correct; verify PostgreSQL service is running

  Issue: "Email not sending"
  Solution: Verify MAIL_* variables; use Gmail App Password, not regular password

  Issue: "OAuth callback mismatch"
  Solution: Update Google Console callback URL to match Render domain

  Issue: "Static files not loading"
  Solution: Gunicorn serves static/ directory; verify files exist

  Issue: "Timeout errors on long requests"
  Solution: Procfile timeout is 120s; increase if needed for long operations
