RENDER DEPLOYMENT STEPS
═══════════════════════════════════════════════════════════════════════════════

1. GITHUB PREPARATION
   - Push to GitHub with all fixes applied
   - Ensure .env is NOT committed (checked via .gitignore)
   - Verify .env.example is committed for reference

2. CREATE RENDER SERVICE
   - Go to https://dashboard.render.com/
   - Click "New +" → "Web Service"
   - Connect to your GitHub repository
   - Select the WealthSync repository

3. SERVICE CONFIGURATION
   Name: wealthsync (or your preferred name)
   Runtime: Python 3
   Build Command: pip install -r requirements.txt
   Start Command: gunicorn app:app
   
4. ENVIRONMENT VARIABLES (set in Render dashboard)
   
   SECRET_KEY=<generate with: python -c "import secrets; print(secrets.token_hex(32))">
   
   GOOGLE_CLIENT_ID=<from Google Cloud Console>
   GOOGLE_CLIENT_SECRET=<from Google Cloud Console>
   
   MAIL_SERVER=smtp.gmail.com
   MAIL_PORT=587
   MAIL_USERNAME=<your-email@gmail.com>
   MAIL_PASSWORD=<app-specific-password-from-Google>
   MAIL_DEFAULT_SENDER=WealthSync <your-email@gmail.com>
   
   PREFERRED_URL_SCHEME=https
   
   Optional (for PostgreSQL):
   DATABASE_URL=postgresql://<user>:<password>@<host>:5432/wealthsync

5. GOOGLE OAUTH CALLBACK SETUP
   - After service is deployed, note your Render URL: https://<service-name>.onrender.com
   - Go to Google Cloud Console → OAuth 2.0 Client IDs
   - Add authorized redirect URI:
     https://<service-name>.onrender.com/auth/google/callback
   
6. DOMAIN SETUP (optional)
   - Go to Render dashboard → Service → Settings
   - Add custom domain or use provided .onrender.com domain
   - Update OAuth callback URL if using custom domain

7. DATABASE CHOICE

   Option A: SQLite (Beta/MVP only)
   ✓ Works immediately, no setup needed
   ✓ Suitable for initial testing with <5 concurrent users
   ✗ Data loss on container restart
   ✗ Not suitable for production with multiple users
   → Use only for beta testing

   Option B: PostgreSQL (Recommended for Production)
   ✓ Data persists across deployments
   ✓ Managed backups by Render
   ✓ Supports unlimited concurrent users
   ✓ Production-ready
   
   To use PostgreSQL:
   1. Create PostgreSQL database on Render
   2. Render will provide DATABASE_URL automatically
   3. Set DATABASE_URL in environment variables
   4. App will automatically detect and use PostgreSQL

8. DEPLOYMENT
   - Click "Deploy"
   - Monitor logs for deployment success
   - Test login, OAuth, and email verification flows

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

Issue: "Module not found" errors
→ Check that all packages in requirements.txt are installed
→ Verify build command is: pip install -r requirements.txt

Issue: Environment variables not set
→ Check Render dashboard → Environment Variables
→ Verify SECRET_KEY, MAIL_*, GOOGLE_* are all set
→ Restart service after changing environment variables

Issue: Google OAuth fails
→ Verify GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET in Render
→ Check callback URL in Google Cloud Console matches Render URL
→ Format: https://<service-name>.onrender.com/auth/google/callback

Issue: Email verification not working
→ Check MAIL_USERNAME and MAIL_PASSWORD in Render
→ Verify email provider settings (Gmail requires App Passwords)
→ Check logs for SMTP errors

Issue: Data loss after restart
→ Indicates SQLite is being used (ephemeral storage)
→ Switch to PostgreSQL for persistence
→ See Database Choice section above
